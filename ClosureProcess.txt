# Job Closing Approval System Enhancement Plan

## Current State Analysis
- Current system: Simple `updateRemarks()` method in `RecruitmentController.php:1053`
- Direct status update to 'closed' with basic remarks only
- Database has approval fields but no structured workflow

## Proposed Multi-Step Approval System

### Step 1: HR Executive Approval
**Trigger:** When HR Executive initiates job closure
**Required Fields:**
- Selected candidate name (who was hired) - select option from employees table
- Replacement of (which employee position is being filled) - select option from employees table
- Job name/position title - taken from employees table based on selected candidate
- Date of joining for selected candidate - taken from employees table based on selected candidate
- Initial assessment notes
- HR Executive approval signature/timestamp

**Status Transition:** `active` → `pending_manager_feedback`
**Access Control:** Only HR Executive (employee_id: 52) can perform this step

### Step 2: Reporting Manager Feedback
**Trigger:** After HR Executive approval
**Required Fields:**
- Team member strengths assessment (text area) - about the employee who joined based on this job listing
- Team member weaknesses assessment (text area) - about the employee who joined based on this job listing
- Should job posting remain open? (Yes/No with reasoning)
- How many people currently in this department? (number)
- Who is the best performer in the team? (employee dropdown)
- Who is the worst performer in the team? (employee dropdown)
- Do you need to find replacement for worst performer? (Yes/No with details)
- Additional comments from reporting manager

**Status Transition:** `pending_manager_feedback` → `partially_closed`
**Access Control:** Department HOD or assigned reporting manager

### Step 3: Final Closure (Automatic)
**Trigger:** After reporting manager feedback submitted
**Actions:**
- Set final status to `closed`
- Set `job_closing_date` to current date
- Generate closure summary report
- Send notification emails to stakeholders

## Database Schema Requirements

### New Table: `rc_job_closure_approvals`
```sql
CREATE TABLE rc_job_closure_approvals (
    id INT PRIMARY KEY AUTO_INCREMENT,
    job_listing_id INT NOT NULL,

    -- HR Executive Step
    selected_candidate_name VARCHAR(255),
    replacement_of VARCHAR(255),
    joining_date DATE,
    hr_assessment_notes TEXT,
    hr_approved_by INT,
    hr_approved_at DATETIME,

    -- Manager Feedback Step
    team_strengths TEXT,
    team_weaknesses TEXT,
    keep_posting_open ENUM('yes','no'),
    keep_posting_reason TEXT,
    current_team_size INT,
    best_performer_id INT,
    worst_performer_id INT,
    need_replacement ENUM('yes','no'),
    replacement_details TEXT,
    manager_comments TEXT,
    manager_approved_by INT,
    manager_approved_at DATETIME,

    -- Status Tracking
    current_step ENUM('hr_approval','manager_feedback','completed'),
    final_closure_date DATETIME,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

### Update Existing Table: `rc_job_listing`
- Add new status values: `pending_manager_feedback`, `partially_closed`
- Keep existing: `active`, `closed`, `draft`

## Controller Implementation

### New Methods in RecruitmentController:

1. **startClosureProcess($jobId)**
   - Initiate closure workflow
   - Create record in `rc_job_closure_approvals`
   - Update job status to `pending_manager_feedback`

2. **hrApprovalForm($jobId)**
   - Display HR approval form
   - Validate HR Executive access
   - Show job details and candidate selection form

3. **submitHrApproval()**
   - Process HR approval form submission
   - Validate required fields
   - Update approval record and job status
   - Send notification to reporting manager

4. **managerFeedbackForm($jobId)**
   - Display manager feedback form
   - Validate manager access (HOD check)
   - Show team member dropdowns for department

5. **submitManagerFeedback()**
   - Process manager feedback form
   - Validate all required assessments
   - Update approval record
   - Trigger final closure

6. **finalClosure($jobId)**
   - Automated final step
   - Update job status to `closed`
   - Generate closure report
   - Send completion notifications

### Modified Method:
- **updateRemarks()**: Add workflow validation, prevent direct closure bypass

## Workflow Logic Implementation

### Status Transitions:
1. `active` → `pending_manager_feedback` (HR approval)
2. `pending_manager_feedback` → `partially_closed` (Manager feedback)
3. `partially_closed` → `closed` (Final closure)

### Access Control Matrix:
- **HR Executive (52)**: Can start closure, submit HR approval
- **Department HOD**: Can submit manager feedback
- **HR Manager (293)**: Can view all closure processes
- **Job Creator**: Can view closure progress only

### Email Notifications:
1. **After HR Approval**: Notify department HOD/reporting manager
2. **After Manager Feedback**: Notify HR Executive and HR Manager
3. **Final Closure**: Notify all stakeholders with closure summary

## Security Considerations

1. **Role Validation**: Strict checking for HR Executive and HOD roles
2. **CSRF Protection**: All forms must include CSRF tokens
3. **Input Sanitization**: Validate and sanitize all text inputs
4. **Audit Logging**: Track all approval actions with timestamps
5. **Status Validation**: Prevent status manipulation and ensure proper workflow sequence

## Benefits of New System

1. **Comprehensive Documentation**: Capture all closure-related information
2. **Management Oversight**: Ensure proper review at multiple levels
3. **Team Assessment**: Valuable feedback on team performance and needs
4. **Audit Trail**: Complete history of closure decisions
5. **Process Standardization**: Consistent closure procedure across all jobs
6. **Performance Insights**: Data on team strengths/weaknesses for future planning
